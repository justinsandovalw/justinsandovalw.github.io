The Field Connection

Objective

Capture real network traffic, document what normal looks like, then identify the signatures of three common attack patterns — port scanning, ARP poisoning, and DNS abuse.

Establishing a Baseline

Before you can detect an attack, you need to know what clean traffic looks like on your network. Key normal patterns I documented:

  • ARP broadcasts limited to local subnet — routine "who has X.X.X.X?" queries
  • DNS queries going only to known resolvers, returning fast, for recognizable domains
  • TCP handshakes completing cleanly — SYN, SYN-ACK, ACK, data, FIN
  • HTTPS dominating outbound — most real traffic is encrypted TLS 1.3

Detecting a Port Scan

Ran a controlled nmap scan from the Kali VM and captured it. The signature is unmistakable once you know what to look for:

# Wireshark display filter — isolate SYN packets (no ACK) tcp.flags.syn == 1 and tcp.flags.ack == 0 # What you see: one source IP, sequential destination ports, # rapid SYN packets, mostly RST responses = textbook nmap

ARP Poisoning Indicators

  • Unsolicited ARP replies (no prior ARP request) — classic sign of gratuitous ARP attack
  • Two different MAC addresses claiming ownership of the same IP
  • ARP reply rate far above baseline — poisoning requires sustained broadcast flooding

DNS Abuse Pattern

  • Queries for long, random-looking subdomains — typical DNS tunneling indicator
  • Unusually large DNS response payloads — data being smuggled in TXT records
  • High query volume to a single domain from one host — exfil over DNS

The Field Connection

Signal tracing in fiber and IPTV work and packet analysis in security are the same mental model: you know what a healthy signal looks like, you watch for deviation, you trace it to source. The tools are different. The thinking isn't.
